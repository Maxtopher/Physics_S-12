Assignment 8: Electronic Output Devices

Use a new output device with a microcontroller integrating at least one input device and avoiding the delay function.

Multitasking the Metro

To practice multitasking on the metroboard, I watched Nathan's recorded lab session from Friday and followed along in the creation of a complex circuit. It contained 3 LEDs wired to separate digital pins and two servo motors each wired to their own digital pins. The goal was to create a program code that could produce 5 simultaneous outputs to each individual component. The key here was to avoid the delay() function, and instead use loops the program can run through very quickly. However, most of the times the code runs through the loop, it will be nullified by an if-statement. By using Millis() for timing, we can control individual components separately and at the same time. I also experimented with class creation in order to make my work more efficient. I really struggled with this part later on, often forgeting to define certain variables in the right place or just not perfectly understanding the setup of the class. However, after playing around with the code, I was eventually able to reproduce the desired result and understand how the code produced it.

The end output for my experimentation phase was three separate LEDs that blinked at differing intervals (and the intervals could be adjusted very quickly in the code with the help of classes) and two servo motors rotating at different speeds.

The LED Button Piano

Circuit Design

For my independent project this week, I decided to integrate the buzzer, buttons and LEDs to create a 5 note breadboard piano. My plan was to have five buttons that could be pressed to each produce a different note on the buzzer. to add a little more complexity, I also wanted a different LED for each button that would also light up when it's note was played. This way, it would be more visually apparent as to which button was being pressed to produce what note.

The circuit I created to meet this idea looks complicated. While there are many wires, it is actually a very simple design. The buzzer is connected to ground and digitalpin 0. Then each button and LED is connected to a separate digitalpin. Every LED is also in series with a 100 Ohm resistor. To save some wiring, the LEDs and buttons share connection to ground.

The follwing Tinkercad design shows the wiring much more clearly.

Coding - Classes

The most challenging part of this assignment was creating the code to make my design work as I intended. I began by just trying to control the lights individually with their corresponding buttons. I did this first with a basic button-LED code, but it felt long and repetitive. Therefore, I tried to use my first self-created class. Once I completed the code and it finally worked, it seemed so simple, however getting there was far from easy. It took lots of trial and error, copy/pasting and experimenting. Finally, I was able to create a class that required only the input of the button pin and LED pin to create functionally paired components. Thankfully, once I got to my functional code, I felt like I had a great understanding of classes and would be able to make another one if necessary.

That was a good thing, since I now had to make another class function for the buzzer. For this code, I needed each button to produce an individual tone from the buzzer only when pressed. That means the tone had to also turn off when the button was unpressed. To do this I intended to make an almost duplicate class while inputing the buzzer tone example in the update() portion. However, I quickly ran into two problems: Without using a strick duration time, I could not turn off the buzzer when it was unpressed. Also, I couldn't figure out how to correctly include the myTone function from class within the class I had created. To solve the second problem, I had to do a little experiementation with variable definitions and eventually was able to integrate the myTone function directly into my class. To solve the problem with the 'duration' I realized if I left the duration on, the tone would end that duration after the button was unpressed. Therefore, if I made the duration very small, it effectively ended with the button being unpressed. The only problem with this was that when two buttons were pressed at once, the buzzer played each tone separately, quickly oscillating back and forth between them (this can be heard at time in the youtube videos below). However, I later discovered that making the delay extremely small (i.e 5 milliseconds) you could optimally hear each note without have the lower note completely overwhelm the other notes entirely. This is the effect if you make the duration 1 millisecond. The class function also allows for very quick changes to the notes played by each button. I set them up to play octaves, randomly varied notes and a short scale.

class ButtonLED
{
    // Class Member Variables
    // These are initialized at startup
    int ledPin;      // the number of the LED pin
    int buttonPin;   // number of button pin

    // These maintain the current state
    int ledState;                   // ledState used to set the LED
    int buttonState;                 // state of button

    // Constructor -
  public:
    ButtonLED(int Bpin, int Lpin)
    {
      buttonPin = Bpin;
      pinMode(buttonPin, INPUT_PULLUP);

      ledPin = Lpin;
      pinMode(ledPin, OUTPUT);
    }

    void Update()
    {
      buttonState = digitalRead(buttonPin);

      if (buttonState == LOW) { 
        digitalWrite(ledPin, HIGH);
      } else {
        digitalWrite(ledPin, LOW);
      }
    }
};

  led5.Update();
}
